Оптимизация локального сервера

10 Июня, 2026

Многие спрашивают, сколько ресурсов нужно для комфортного Homelab-а. Недавно я анализировал нагрузку на свой сервер и хочу развеять один популярный миф.

При мониторинге системы можно заметить цифру в 12 ГБ оперативной памяти. Важно понимать: это не предел системы, а лишь объем, который потребляют сервисы на данный момент. Архитектура настроена таким образом, чтобы динамически выделять ресурсы под конкретные нужды, будь то виртуальные машины или тяжелые docker-контейнеры.

Основные запущенные сервисы:

  • Локальный Wiki-портал для документации
  • Личное Cloud-хранилище
  • Proxy-сервер для маршрутизации трафика
  • Кластер Kubernetes (Minikube / k3s)

Правильная настройка лимитов в K8s позволяет держать аптайм даже при пиковых нагрузках во время тестирования скриптов на Python.

Проект: Нечеткая логика на Python

15 Января, 2026

В рамках лабораторной работы была поставлена интересная задача — использовать библиотеку scikit-fuzzy для моделирования реальной ситуации.

Модель оценивает шансы игрока попасть в баскетбольную команду на основе нескольких входных параметров (рост, процент попаданий, выносливость). Использование алгоритмов Мамдани и Сугено показало отличные результаты при настройке правил (rules) для нечеткого вывода.

import skfuzzy as fuzz
from skfuzzy import control as ctrl

# Пример инициализации антецедентов
height = ctrl.Antecedent(np.arange(160, 220, 1), 'height')
stamina = ctrl.Antecedent(np.arange(0, 101, 1), 'stamina')

В итоге система выдает скоринг в процентах, который помогает тренеру принять объективное решение.
